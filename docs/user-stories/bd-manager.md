# User Story: Business Development Manager

## Persona: Marcus Johnson

**Role**: Business Development Manager
**Company Size**: 150 employees (mid-size government contractor)
**Experience**: 12 years in government contracting
**Certifications**: SDVOSB, HUBZone
**Goal**: Increase win rate and efficiency of opportunity qualification

---

## User Journey

### 1. Efficient Opportunity Qualification

**Scenario**: Marcus needs to quickly assess large volumes of new opportunities daily.

**Actions**:
1. Logs into GovAI dashboard
2. Views statistics summary:
   - 45 new opportunities matching NAICS codes
   - 12 AI-recommended BIDs
   - 8 requiring RESEARCH
3. Filters evaluations by recommendation type
4. Reviews BID recommendations first (highest priority)
5. Scans fit scores and win probabilities
6. Opens detailed analysis for top candidates

**Acceptance Criteria**:
- Dashboard loads with real-time statistics
- One-click filtering by recommendation
- Sort by fit score or win probability
- Bulk review capability
- Time to qualify opportunity reduced by 80%

---

### 2. Deep Dive Analysis

**Scenario**: Marcus needs to present qualified opportunities to leadership with justification.

**Actions**:
1. Selects high-potential opportunity (Fit: 92%, Win Prob: 75%)
2. Reviews comprehensive AI analysis:
   - **Strengths**: SDVOSB set-aside, relevant past performance, geographic preference match
   - **Weaknesses**: Short response timeline, new agency relationship
   - **Key Requirements**: SECRET clearance, 5-year performance history, AWS certification
   - **Risk Factors**: Incumbent advantage, competitive pricing expected
3. Adds personal notes with additional context
4. Saves to pipeline as "BIDDING"
5. Exports analysis for leadership presentation

**Acceptance Criteria**:
- Complete AI analysis available on single page
- Notes field supports markdown formatting
- Pipeline status easily updated
- Analysis explains AI reasoning clearly
- Historical context available for repeat opportunities

---

### 3. Pipeline Management Across Team

**Scenario**: Marcus manages a portfolio of opportunities at various stages.

**Actions**:
1. Opens Pipeline view (Kanban board)
2. Reviews opportunities by status:
   - WATCHING: 15 opportunities (monitoring)
   - BIDDING: 8 opportunities (active proposals)
   - WON: 3 (this quarter)
   - LOST: 2 (this quarter)
3. Views win rate: 60%
4. Drags opportunity from WATCHING to BIDDING
5. Updates notes with proposal status
6. Archives PASSED opportunities

**Acceptance Criteria**:
- Visual Kanban board layout
- Drag-and-drop status changes
- Win rate automatically calculated
- Notes and history preserved
- Filter by date range

---

### 4. Strategic Decision Making

**Scenario**: Marcus needs to prioritize limited BD resources.

**Actions**:
1. Reviews statistics dashboard
2. Analyzes recommendation distribution:
   - BID: 25% of evaluated opportunities
   - RESEARCH: 30% need more analysis
   - NO_BID: 45% not recommended
3. Compares average fit scores across NAICS codes
4. Identifies highest-performing opportunity categories
5. Adjusts company profile to refine matching
6. Re-evaluates RESEARCH opportunities after profile update

**Acceptance Criteria**:
- Statistical insights readily available
- Trend analysis over time
- NAICS code performance comparison
- Profile adjustments trigger re-evaluation
- Export data for reports

---

### 5. Proactive Opportunity Monitoring

**Scenario**: Marcus wants to stay ahead of competition.

**Actions**:
1. Configures real-time email notifications
2. Receives immediate alert for high-value BID recommendations
3. Sets up deadline reminders:
   - 7 days before: Initial alert
   - 3 days before: Preparation reminder
   - 1 day before: Final reminder
4. Uses manual discovery trigger for on-demand updates
5. Tracks new opportunities throughout the day

**Acceptance Criteria**:
- Configurable notification frequency
- Immediate alerts for high-priority matches
- Multiple reminder touchpoints
- Manual refresh capability
- Mobile-friendly notifications
